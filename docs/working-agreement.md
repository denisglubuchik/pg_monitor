# Working Agreement

Обязательные правила работы над проектом:

1. Перед каждой итерацией сначала составляется детальный план итерации.
2. Реализация всегда идет в порядке: тесты -> код.
3. Любые архитектурные или технические решения сначала согласуются.
4. Любые неясности или недостающий контекст сначала уточняются у владельца проекта.
5. Если решение не согласовано, реализация не начинается.

## Definition of Ready для итерации

Итерация может стартовать только если:
- согласован детальный план;
- согласованы критерии готовности;
- нет неуточненных вопросов.

## Definition of Done для итерации

Итерация считается завершенной только если:
- сначала написаны и пройдены тесты;
- код реализован согласно согласованному плану;
- результат подтвержден по согласованным критериям готовности.
